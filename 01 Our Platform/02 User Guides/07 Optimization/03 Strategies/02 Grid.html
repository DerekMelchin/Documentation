What is grid search:
- looks through all combinations of parameters.
-Searches the cartesian product of the parameter value sets you provide (start, stop, step)
- <display a gif that shows a table of values and how the optimizer just works its way through all of the cells>

Why use grid search (pros):
-Tests all possible combination of parameters (Brute force)
-Currently the only option available in the optimization wizard

Why not to use grid search (cons):
-Can be an expensive option since it tests all parameter combinations
-Curse of dimensionality
-There is no intelligent decision being made on what parameters to test next (which other search strategies do)
